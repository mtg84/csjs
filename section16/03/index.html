JavaScript Is Single Threaded!

At any given point in time, that single JS thread is running at most one line of JS code.

How Asynchronous Callbacks Actually work?

THE BROWSER DOES THE WORK

The browser is separate from JS

How?

Browsers come with Web APIs that are able to handle certain tasks in the background
(like making requests or setTimeout)

The JS call stack recognizes these Web API functions and passes them off to the browser to take care of

Once the browser finishes those tasks, they return and are pushed on to the stack as a callback
